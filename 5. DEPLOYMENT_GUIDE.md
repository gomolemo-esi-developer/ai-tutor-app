## PRE-DEPLOYMENT CHECKLIST

Before starting, verify:

- [ ] Git installed and initialized
- [ ] Docker installed locally
- [ ] GitHub account

## STEP 1: LOCAL CLEANUP

### 1.1 Run Cleanup Script (Bash/Linux/Mac)

```bash
bash CLEANUP_SCRIPT.sh
```

This will:
- Delete `node_modules/` directories
- Delete build artifacts (`dist/`, `__pycache__/`)
- Delete Python virtual environment
- Delete local database files
- Delete Windows batch scripts
- Delete temporary data files

### 1.2 Manual Cleanup (Windows PowerShell)

If not using bash, manually delete:

```powershell
# Backend
Remove-Item -Recurse -Force "ai-tutor-app\backend\dist"
Remove-Item -Recurse -Force "ai-tutor-app\backend\node_modules"
Remove-Item -Force "ai-tutor-app\backend\backend.log"
Remove-Item -Force "ai-tutor-app\backend\lambda-code.tar.gz"
Remove-Item -Force "ai-tutor-app\backend\lambda-code.zip"

# Frontend
Remove-Item -Recurse -Force "ai-tutor-app\tutorverse-hub-main\dist"
Remove-Item -Recurse -Force "ai-tutor-app\tutorverse-hub-main\node_modules"

# RAG
Remove-Item -Recurse -Force "RAG18Nov2025-1\venv"
Remove-Item -Recurse -Force "RAG18Nov2025-1\__pycache__"
Remove-Item -Recurse -Force "RAG18Nov2025-1\chroma_db"
Remove-Item -Force "RAG18Nov2025-1\*.bat"
Remove-Item -Force "RAG18Nov2025-1\educator_files_list.json"
Remove-Item -Force "RAG18Nov2025-1\*_summary.json"
Remove-Item -Force "RAG18Nov2025-1\requirements_full.txt"
Remove-Item -Force "RAG18Nov2025-1\requirements_simple.txt"
```

### 1.3 Update .gitignore

Add to root `./.gitignore`:

```
# Environment files
.env
.env.backend
.env.frontend
.env.rag
.env.local
.env.local.example

# Build artifacts
dist/
build/
*.egg-info/

# Dependencies
node_modules/
venv/
env/
.venv

# Python cache
__pycache__/
*.pyc

# Logs
*.log
backend.log

# Local development
.DS_Store
Thumbs.db
*.swp
*.swo

# Docker
.dockerignore
```

### 1.4 Remove .env Files from Git History

```bash
# Stop tracking .env files
git rm --cached ai-tutor-app/backend/.env 2>/dev/null || true
git rm --cached RAG18Nov2025-1/.env 2>/dev/null || true

# Commit changes
git add .gitignore
git commit -m "Update .gitignore for production deployment"
```

## STEP 2: ENVIRONMENT CONFIGURATION

### 2.1 Create Local .env Files

**For Backend** (`ai-tutor-app/backend/.env`):

Copy from `.env.example`:
```bash
cp ai-tutor-app/backend/.env.example ai-tutor-app/backend/.env
```

Edit with actual values:
```
NODE_ENV=production
AWS_REGION=us-east-2
AWS_ACCOUNT_ID=your_account_id
ENVIRONMENT=production
RAG_SERVICE_URL=http://rag-service:8000
RAG_ENABLE=true
RAG_TIMEOUT=30000
RAG_RETRY_ATTEMPTS=3
# Add other required env vars
```

**For RAG Service** (`RAG18Nov2025-1/.env`):

Create `.env` with required variables. Check `config.py` for requirements:
```
OPENAI_API_KEY=sk-...
ENVIRONMENT=production
PYTHONUNBUFFERED=1
# Add other required vars
```

**For Frontend** (`ai-tutor-app/tutorverse-hub-main/.env`):

Create `.env` with backend API URL:
```
VITE_API_URL=http://localhost:3001
VITE_ENVIRONMENT=production
```

### 2.2 Verify Docker Configuration

Ensure `docker-compose.yml` has correct settings:

```yaml
# Check these sections exist:
services:
  frontend:
    ports:
      - "3000:3000"  # Frontend on 3000
  
  backend:
    ports:
      - "3001:3000"  # Backend on 3001 (internal 3000)
    environment:
      - RAG_SERVICE_URL=http://rag-service:8000  # Important!
  
  rag-service:
    ports:
      - "8000:8000"  # RAG on 8000
    volumes:
      - chroma_data:/app/chroma_db  # Persistent storage
```

## STEP 3: LOCAL TESTING

### 3.1 Build Docker Images

```bash
docker-compose build
```

Expected output:
```
Building frontend ... done
Building backend ... done
Building rag-service ... done
```

### 3.2 Run Locally

```bash
docker-compose up
```

Wait for all services to start. You should see:
```
tutorverse-frontend | Accepting connections at http://localhost:3000
tutorverse-backend | Server running on port 3000
tutorverse-rag | Uvicorn running on 0.0.0.0:8000
```

### 3.3 Test Endpoints

Open browser or use `curl`:

```bash
# Frontend (should return HTML)
curl http://localhost:3000

# Backend health check
curl http://localhost:3001/health

# RAG health check
curl http://localhost:8000/health

# RAG docs (should work)
curl http://localhost:8000/docs
```

### 3.4 Test API Calls

Test a real API call from frontend to backend:
- Open browser to http://localhost:3000
- Open developer console (F12)
- Check Network tab
- Verify requests to `http://localhost:3001` work

### 3.5 Stop Local Services

```bash
docker-compose down
```

## STEP 5: DEPLOY TO RENDER.COM

### 5.1 Create Render Account & Connect GitHub

1. Go to [render.com](https://render.com)
2. Sign up with GitHub
3. Authorize Render to access your repositories

### 5.2 Deploy Frontend Service

1. Click "New +" → "Web Service"
2. Select your GitHub repository
3. Fill in:
   - **Name**: `tutorverse-frontend`
   - **Region**: `Oregon` (or closest to users)
   - **Branch**: `main`
   - **Runtime**: `Docker`
   - **Build Command**: `docker-compose build --no-cache`
   - **Start Command**: `docker-compose up frontend`
4. Click "Advanced" and add:
   - **Docker Compose File**: `/docker-compose.yml`
5. Create Web Service

### 5.3 Deploy Backend Service

1. Click "New +" → "Web Service"
2. Select same GitHub repository
3. Fill in:
   - **Name**: `tutorverse-backend`
   - **Region**: `Oregon`
   - **Branch**: `main`
   - **Runtime**: `Docker`
   - **Build Command**: `docker-compose build --no-cache`
   - **Start Command**: `docker-compose up backend`
4. Click "Advanced":
   - Add environment variables from `.env.backend`:
     ```
     NODE_ENV=production
     AWS_REGION=us-east-2
     RAG_SERVICE_URL=http://tutorverse-rag:8000
     # ... other vars
     ```
5. Create Web Service

### 5.4 Deploy RAG Service

1. Click "New +" → "Background Worker"
2. Select same GitHub repository
3. Fill in:
   - **Name**: `tutorverse-rag`
   - **Region**: `Oregon`
   - **Branch**: `main`
   - **Runtime**: `Docker`
   - **Build Command**: `docker-compose build --no-cache`
   - **Start Command**: `docker-compose up rag-service`
4. Click "Advanced":
   - Add environment variables from `.env.rag`:
     ```
     OPENAI_API_KEY=sk-...
     ENVIRONMENT=production
     PYTHONUNBUFFERED=1
     # ... other vars
     ```
   - **Add Persistent Disk**: 
     - Path: `/app/chroma_db`
     - Size: 10GB (recommended)
5. Create Background Worker

### 5.5 Update Service URLs

After all services deploy, update configuration:

1. Get service URLs from Render dashboard
   - Frontend URL: `https://tutorverse-frontend.onrender.com`
   - Backend URL: `https://tutorverse-backend.onrender.com`
   - RAG URL: `https://tutorverse-rag.onrender.com`

2. Update Backend environment variable:
   - `RAG_SERVICE_URL=https://tutorverse-rag.onrender.com`

3. Update Frontend environment variable:
   - `VITE_API_URL=https://tutorverse-backend.onrender.com`

4. Redeploy services:
   - Go to each service
   - Click "Manual Deploy"
   - Or wait for auto-redeploy

## POST-DEPLOYMENT VALIDATION

### 6.1 Verify Services Are Running

Check Render dashboard:
- [ ] Frontend shows "Live"
- [ ] Backend shows "Live"
- [ ] RAG shows "Live"

### 6.2 Check Logs

For each service:
1. Click on service name
2. View "Logs" tab
3. Should see startup logs, no errors

Look for:
- `Server running on port 3000` (backend)
- `Accepting connections at http://localhost:3000` (frontend)
- `Uvicorn running on 0.0.0.0:8000` (RAG)

### 6.3 Test Health Endpoints

```bash
# Backend health
curl https://tutorverse-backend.onrender.com/health

# RAG health
curl https://tutorverse-rag.onrender.com/health
```

Should return `200 OK`.

### 6.4 Test API Calls

1. Open https://tutorverse-frontend.onrender.com
2. Open browser DevTools (F12)
3. Go to Network tab
4. Interact with app
5. Verify API calls to backend succeed

### 6.5 Test RAG Integration

Send request to backend that uses RAG:
```bash
curl -X POST https://tutorverse-backend.onrender.com/api/rag-query \
  -H "Content-Type: application/json" \
  -d '{"query": "test query"}'
```

Should connect to RAG service and return response.

## TROUBLESHOOTING

### Issue: Frontend builds but shows blank page

**Causes**:
- API URL in frontend points to wrong backend
- CORS configuration needed on backend

**Fix**:
1. Check browser console for errors
2. Update `VITE_API_URL` in frontend environment
3. Add CORS headers in backend (if using Express):
   ```typescript
   app.use(cors({
     origin: process.env.FRONTEND_URL || '*',
     credentials: true
   }));
   ```

### Issue: Backend health check fails

**Causes**:
- No `/health` endpoint
- Endpoint returns wrong status code
- Port configuration wrong

**Fix**:
1. Add health endpoint to backend:
   ```typescript
   app.get('/health', (req, res) => {
     res.status(200).json({ status: 'ok' });
   });
   ```
2. Check logs for specific error
3. Verify environment variables set correctly

### Issue: RAG service won't start

**Causes**:
- Missing system dependencies (ffmpeg)
- Missing Python dependencies
- Missing environment variables
- No `/health` endpoint

**Fix**:
1. Check logs for specific error
2. Verify `requirements.txt` is complete:
   ```bash
   pip install -r RAG18Nov2025-1/requirements.txt
   ```
3. Add health endpoint to `main.py`:
   ```python
   @app.get("/health")
   async def health():
       return {"status": "ok"}
   ```
4. Verify all required env vars set in Render

### Issue: Chroma database not persisting

**Causes**:
- Persistent disk not mounted
- Wrong path in mount

**Fix**:
1. Check RAG service has persistent disk mounted
2. Verify mount path is `/app/chroma_db`
3. Check logs to see where data actually stored

### Issue: Services can't communicate

**Causes**:
- Wrong internal service URLs
- Services not on same network
- Firewall blocking

**Fix**:
1. Use internal service names in docker-compose URLs
2. Backend should use: `http://rag-service:8000`
3. In Render, services can't communicate - use external URLs

**Important**: In Render.com, internal service names don't work. Use external URLs:
- Backend → RAG: Use `https://tutorverse-rag.onrender.com`
- Frontend → Backend: Use `https://tutorverse-backend.onrender.com`

Update `docker-compose.yml` for production:
```yaml
services:
  backend:
    environment:
      - RAG_SERVICE_URL=https://tutorverse-rag.onrender.com  # External URL
```

### Issue: Out of memory or slow performance

**Causes**:
- Too many dependencies
- Large ML models loaded
- Inefficient queries

**Fix**:
1. Check service resource usage in Render
2. Consider upgrading plan
3. Optimize Python dependencies (remove unused)
4. Cache RAG responses if possible

## MONITORING & MAINTENANCE

### Monitor Logs Daily

```bash
# In Render dashboard
# For each service: Logs tab
```

Look for:
- Error messages
- Performance issues
- Memory/CPU spikes

### Update Environment Variables

If API keys or configuration changes:
1. Go to service settings
2. Edit environment variables
3. Click "Save"
4. Service auto-redeploys

### Redeploy Services

When pushing code changes:
```bash
git push origin main
```

Render will auto-detect and redeploy, OR manually:
1. Go to service
2. Click "Manual Deploy"
3. Click "Deploy Latest Commit"

### Backup Data

Chroma database should be backed up:
1. Use Render backup tool for persistent disk
2. Or periodically export data from RAG service

## ROLLBACK PLAN

If deployment fails:

1. **Check service logs** - Identify the issue
2. **Fix and commit** - Push corrected code
3. **Manual redeploy** - Click "Manual Deploy" in Render
4. **Or rollback commit** - Use previous working version

To rollback:
```bash
git revert HEAD  # Revert last commit
git push origin main
# Service will auto-redeploy
```
