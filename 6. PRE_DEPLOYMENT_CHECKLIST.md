## PHASE 6: LOCAL TESTING

### Build Test
- [ ] `docker-compose build` completes successfully
  - [ ] Frontend image builds
  - [ ] Backend image builds
  - [ ] RAG image builds
- [ ] No warnings or errors in build output

### Runtime Test
- [ ] `docker-compose up` starts all services
  - [ ] Frontend starts without errors
  - [ ] Backend starts without errors
  - [ ] RAG starts without errors
- [ ] All health checks pass

### Functionality Test
- [ ] Frontend loads at `http://localhost:3000`
  - [ ] HTML page renders
  - [ ] CSS/JavaScript loads
  - [ ] No console errors

- [ ] Backend health check: `curl http://localhost:3001/health`
  - [ ] Returns 200 OK
  - [ ] Response is valid JSON

- [ ] RAG health check: `curl http://localhost:8000/health`
  - [ ] Returns 200 OK
  - [ ] Response is valid JSON

- [ ] RAG documentation: `curl http://localhost:8000/docs`
  - [ ] Swagger UI loads
  - [ ] API endpoints documented

### Integration Test
- [ ] Frontend can reach backend
  - [ ] Open http://localhost:3000
  - [ ] Open DevTools (F12) → Network tab
  - [ ] Interact with app that makes API calls
  - [ ] Verify requests to `http://localhost:3001` succeed
  - [ ] No CORS errors in console

- [ ] Backend can reach RAG (if applicable)
  - [ ] Make API call that triggers RAG
  - [ ] Verify response from RAG received

### Shutdown Test
- [ ] `docker-compose down` stops all services cleanly
- [ ] Verify `docker ps` shows no running containers
