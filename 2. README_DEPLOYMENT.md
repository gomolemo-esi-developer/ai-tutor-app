
**Fix**: Run `bash CLEANUP_SCRIPT.sh`

### Issue #2: .env Files Tracked in Git
**Impact**: 🔴 SECURITY RISK - API keys exposed
```bash
# These are in git but shouldn't be:
ai-tutor-app/backend/.env
RAG18Nov2025-1/.env
```

**Fix**:
```bash
git rm --cached ai-tutor-app/backend/.env
git rm --cached RAG18Nov2025-1/.env
echo ".env*" >> .gitignore
git add .gitignore
git commit -m "Remove secrets from tracking"
```

### Issue #3: No Health Endpoints
**Impact**: Services won't deploy (health checks fail)

**Fix**: Verify these exist:
- Backend: `npm run health` script in package.json
- RAG: `@app.get("/health")` in main.py

## 🚨 COMMON MISTAKES TO AVOID

| Mistake | Impact | How to Avoid |
|---------|--------|--------------|
| Push `.env` to GitHub | 🔴 Security breach | Use `.gitignore` |
| Forget health endpoints | 🔴 Deployment fails | Add `/health` route |
| Wrong service URLs | 🔴 Services can't communicate | Use internal names locally, external in prod |
| Large git repo | 🟡 Slow cloning | Run cleanup script |
| Test only in Docker | 🟡 Missing bugs | Test locally first |
| No error monitoring | 🟡 Can't debug issues | Set up Sentry |
